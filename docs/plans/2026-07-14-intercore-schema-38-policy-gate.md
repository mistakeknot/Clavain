# Intercore Schema 38 Policy Gate Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use clavain:executing-plans to implement this plan task-by-task.

**Bead:** `Sylveste-vyn`

**Goal:** Restore authorized `.beads/push.sh` operation on Intercore schema 38 without weakening signer, ledger, policy, or signed-record enforcement.

**Architecture:** Clavain will define one audited authorization-ledger compatibility range in Go: schemas 36 through 38. Migrations 37 and 38 add unrelated feedback and agency-event tables, so the signed `authorizations` contract remains the schema-36 contract. The shell gate will validate the same closed range and continue to reject older or unreviewed future schemas.

**Tech Stack:** Go, SQLite, Bash, jq, Bats, Beads/Dolt.

## Must-Haves

**Truths**
- `policy doctor --require-signer` accepts a valid schema-38 authorization domain.
- Schema 38 still requires the private signer key and a fully verified ledger.
- Schemas below 36 and above 38 remain unsupported.
- `bd-push-dolt.sh` runs policy check and signed decision recording before invoking Dolt.
- The real Sylveste `.beads/push.sh` completes on the deployed schema-38 database.

**Artifacts**
- `cmd/clavain-cli/authz_sign.go` owns the audited Go compatibility contract.
- `scripts/gates/_common.sh` enforces the same contract on doctor output.
- Go and Bats tests cover supported and unsupported versions plus enforcement ordering.

**Key Links**
- `.beads/push.sh` calls `scripts/gates/bd-push-dolt.sh`.
- `bd-push-dolt.sh` calls `gate_require_signer`, then `gate_check`, then `gate_record_signed`, then Dolt.
- `gate_require_signer` trusts only a successful, well-formed `policy doctor` response inside the audited schema range.
